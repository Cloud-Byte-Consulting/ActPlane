\section{Introduction}

% ¶1: Context — agents are powerful and widely deployed
AI coding agents operate as long-running processes with access to shells,
source trees, package managers, and external APIs~\cite{agentsight}. They
autonomously write code, run tests, and manage infrastructure across
extended sessions.

% ¶2: Agents need harnesses; instruction files are behavioral policy
As agents gain autonomy, they require \emph{harnesses} that apply
behavioral policies---rules that govern what the agent may do, in what
order, and under what conditions~\cite{langchain-harness,fowler-harness}.
Projects encode such policies in instruction files (\texttt{CLAUDE.md},
\texttt{AGENTS.md})~\cite{chatlatanagulchai2025manifests,lulla2026agentsmd}.

% ¶3: Empirical — the problem is pervasive; LLMs can't self-enforce
These policies are pervasive and demand deterministic enforcement. Our
empirical study of 64~projects (\S\ref{sec:empirical}) finds that 64\%
of instruction-file statements are behavioral directives, and 83\% of
those involve system-level behavior---commands executed, files accessed,
and network connections made. Because LLM compliance is inherently
probabilistic~\cite{liu2024lost}, policy cannot reside inside the model
alone: even cooperative agents routinely violate declared constraints
through planning errors, shell-script side effects, and opaque tool
behavior.

% ¶4: Semantic gap — existing deterministic mechanisms also fail
Yet existing harnesses fail to connect these intent-level declarations
to deterministic system-level enforcement, leaving a \emph{semantic
gap}. Tool-call
guardrails~\cite{wang2025agentspec,shi2025progent} lose track of
system-level actions when agents shell out or spawn subprocesses: the
same \texttt{git~push} can be reached through a tool call, a
\texttt{bash~-c} wrapper, or a compiled binary---and 81\% of projects
require cross-event state tracking that tool-level interception cannot
provide. OS-level
mechanisms~\cite{ciliumtetragon,flume,pasquier2018camquery} observe all
system effects but expect pre-defined static policy and return only
system events without semantic context; yet 74\% of system-level rules
require intent-level project or task context that only the agent
possesses.

% ¶5: Insight — agent authors policy, OS enforces it
This gap arises because no existing layer lets the agent---the only
entity with the context to instantiate its own rules---also program the
OS-level mechanism that enforces them. Since 
only the agent itself knows the project and task context, it must author its own
enforcement policy but must not enforce it: trusted to honestly declare
intent, but not trusted to reliably follow it during execution. This
separation calls for a programmable interface that lets agents
dynamically define and adapt policy for their own system-level actions
based on their intent context.

% ¶5: System — \sys{}
We realize this in \sys{}. Agents and developers declare behavioral
policies in a compact DSL that agents themselves can generate from
natural-language directives; \sys{} compiles them into labeled
information-flow rules and loads them into an eBPF/BPF-LSM enforcement
engine. Named labels propagate across processes, files, and network
endpoints at kernel hooks, encoding execution history as checkable state.
When a labeled event matches a rule, \sys{} returns semantic
feedback---which rule matched, why, and what compliant alternative
satisfies it. Each rule independently selects one of three effects:
notify (observe), block (pre-operation denial), or kill (post-operation
termination). A layered authority model ensures that higher-authority
rules (e.g., platform-level ``do not leak secrets'') cannot be weakened by
agent-authored policy.

We make three contributions:

\begin{enumerate}
\item An \textbf{empirical study} of instruction files from 64~projects,
showing that cross-event information-flow policies are pervasive (81\% of
repositories) and that 74\% of system-level directives require project or
task context that only the agent can supply (\S\ref{sec:empirical}).

\item \textbf{\sys{}}, a programmable OS-level enforcement layer for agent
harnesses. \sys{} compiles behavioral policies into eBPF/BPF-LSM labeled
information-flow rules, enforces them across process, file, and network
boundaries, and returns semantic feedback that reconnects rule matches to
intent-level remediation (\S\ref{sec:design}--\ref{sec:implementation}).

\item An \textbf{evaluation} across 607~directive-derived policies,
system workloads, and a 20-task OctoBench subset. \sys{} improves
policy compliance over prompt-only and tool-regex baselines while adding
less than 2\% end-to-end overhead on agent workloads
(\S\ref{sec:evaluation}).
\end{enumerate}
